- <span data-ttu-id="4e737-128">**고속 가리키기.**</span><span class="sxs-lookup"><span data-stu-id="4e737-128">**High speed pointing.**</span></span> <span data-ttu-id="4e737-129">안과는 인체에서 가장 빠르게 반응하는 반응형 반응형입니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-129">The eye muscle is the fastest reacting muscle in the human body.</span></span>

<span data-ttu-id="4e737-161">이는 사용자가 눈동자 대상을 사용 하 여 응용 프로그램에 너무 작은 대상을 선택할 때 사용자에 게 호출할 수 있는 느낌입니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-161">This is a feeling you might invoke in your users when forcing them to select targets that are too small in your application using eye targeting.</span></span>
<span data-ttu-id="4e737-162">사용자를 위해 기분 좋고 편리한 환경을 디자인하려면 대상에 대한 시각적 각도를 2° 이상으로 유지하는 것이 좋습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-162">For your design, to create a pleasant and comfortable experience for your users, we recommend that targets should be at least 2° in visual angle, preferably larger.</span></span>

- <span data-ttu-id="4e737-166">**안정성 추적:** 눈이 새 조건에 맞게 조정 될 때 눈 추적 정확도는 약간의 변화를 덜 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-166">**Tracking reliability:** Eye tracking accuracy may degrade a little in changing light as your eyes adjust to the new conditions.</span></span>
<span data-ttu-id="4e737-167">이는 반드시 응용 프로그램 디자인에 영향을 주지는 않지만, 정확성은 2 ° 제한 이내에 있으므로 사용자가 다시 보정 해야 할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-167">While this shouldn't necessarily affect your application design, as the accuracy should be within the 2° limitation, it may be necessary for the user to calibrate again.</span></span> 


## <a name="design-recommendations"></a><span data-ttu-id="4e737-168">디자인 권장 사항</span><span class="sxs-lookup"><span data-stu-id="4e737-168">Design recommendations</span></span>
<span data-ttu-id="4e737-169">다음은 눈에 잘 맞는 입력에 대해 설명 된 장점과 문제에 따라 특정 디자인 권장 사항 목록입니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-169">The following is a list of specific design recommendations based on the described advantages and challenges for eye-gaze input:</span></span>

1. <span data-ttu-id="4e737-170">**눈동자-응시는 Head-응시와 동일 하지 않습니다.**</span><span class="sxs-lookup"><span data-stu-id="4e737-170">**Eye-gaze isn't the same as Head-gaze:**</span></span>
    - <span data-ttu-id="4e737-171">**빠르고 비정형 시선 이동이 입력 작업에 적합한지 여부를 고려합니다.** 빠르고 비정형 시선 이동은 보기 영역에서 대상을 빠르게 선택하는 데 뛰어나지만 부드러운 입력 비동기(예: 주석 그리기 또는 둘러싸기)가 필요한 작업에는 덜 적용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-171">**Consider whether fast yet ragged eye movements fit your input task:** While our fast and ragged eye movements are great at quickly selecting targets across our field of view, it's less applicable for tasks that require smooth input trajectories (for example, drawing or encircling annotations).</span></span> <span data-ttu-id="4e737-172">이 경우 손 또는 머리 가리키기가 더 나을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-172">In this case, hand or head pointing should be preferred.</span></span>
  
    - <span data-ttu-id="4e737-173">**사용자의 시선 응시에 직접 연결하지 않습니다(예: 슬라이더 또는 커서).**</span><span class="sxs-lookup"><span data-stu-id="4e737-173">**Avoid attaching something directly to the user’s eye-gaze (for example, a slider or cursor).**</span></span>
<span data-ttu-id="4e737-174">커서를 통해 프로젝트된 시선 응시 신호의 약간의 오프셋으로 인해 "커서를 클리핑하는" 효과가 발생할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-174">With cursors, this may result in a "fleeing cursor" effect because of slight offsets in the projected eye-gaze signal.</span></span> <span data-ttu-id="4e737-175">슬라이더를 사용하면 개체가 올바른 위치에 있는지 확인하는 동시에 눈과 슬라이더를 제어하는 이중 역할과 충돌할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-175">With a slider, it can conflict with the double role of controlling the slider with your eyes while also wanting to check whether the object is at the correct location.</span></span> <span data-ttu-id="4e737-176">슬라이더의 예에서는 손 제스처와 함께 시선 응시를 사용하는 것이 더 적합합니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-176">For the example of the slider, it makes more sense to use eye-gaze in combination with hand gestures.</span></span> <span data-ttu-id="4e737-177">즉, 사용자는 여러 슬라이더 간에 빠르고 쉽게 전환하여 손을 들어 엄지 단추와 인덱스 손가락을 모으기만 하면 슬라이더를 잡고 이동할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-177">This means that the user could quickly and effortlessly switch among many sliders, raising up their hand and pinching their thumb and index finger to grab and move it.</span></span> <span data-ttu-id="4e737-178">손가락 모으기가 해제되면 슬라이더의 이동이 중지됩니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-178">When the pinch is released, the slider stops moving.</span></span> <span data-ttu-id="4e737-179">사용자는 특히 신호가 해당 사용자에게 부정확한 경우 과부하고 방해가 될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-179">Users could become overwhelmed and distracted, especially if the signal is imprecise for that user.</span></span> 
  
2. <span data-ttu-id="4e737-180">**시선 응시를 다른 입력과 결합:** 손 제스처, 음성 명령 또는 단추 누른 것과 같은 다른 입력과 시선 추적을 통합하면 다음과 같은 몇 가지 이점이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-180">**Combine eye-gaze with other inputs:** The integration of eye tracking with other inputs, such as hand gestures, voice commands or button presses, provides several advantages:</span></span>
    - <span data-ttu-id="4e737-181">**무료 관찰 허용:** 눈의 주된 역할이 환경을 관찰하는 것임을 고려할 때, 피드백이나 작업(시각적 개체, 감사자 등)을 트리거하지 않고도 사용자가 주변을 둘러볼 수 있도록 허용하는 것이 중요합니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-181">**Allow for free observation:** Given that the main role of our eyes is to observe our environment, it's important that users are allowed to look around without triggering any (visual, auditory, and so on) feedback or actions.</span></span> 
    <span data-ttu-id="4e737-182">시선 추적을 다른 입력 컨트롤과 결합하면 시선 추적 관찰과 입력 컨트롤 모드 간에 원활하게 전환할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-182">Combining eye tracking with another input control allows smooth transitioning between eye tracking observation and input control modes.</span></span>
  
    - <span data-ttu-id="4e737-183">**강력한 컨텍스트 공급자:** 음성 명령을 말하거나 손 제스처를 사용하여 사용자가 보고 있는 위치 및 내용에 대한 정보를 사용하면 입력을 보기의 필드에 원활하게 채널링할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-183">**Powerful context provider:** Using information about where and what the user is looking at while saying a voice command or using a hand gesture allows seamlessly channeling the input across the field-of-view.</span></span> <span data-ttu-id="4e737-184">예를 들어, 대상과 의도 한 대상을 확인 하 여 운용을 신속 하 고 신속 하 게 선택 하 여 장면 전체에 배치 하 _는 것을 말합니다._</span><span class="sxs-lookup"><span data-stu-id="4e737-184">For example: Say _"put that there"_ to quickly and fluently select and position a hologram across the scene by looking at a target and its intended destination.</span></span>

    - <span data-ttu-id="4e737-196">사용자가 가장 작은 대상을 선택 하도록 강요 하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-196">Don't force users to select tiny targets.</span></span> <span data-ttu-id="4e737-197">대상이 충분히 크고 시스템이 잘 설계 된 경우, 사용자는 자신에 게 자신을 간편 하 게 설명 하 고 마법을 설명 합니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-197">Research has shown that if targets are sufficiently large and system are designed well, users describe their interactions as effortless, and magical.</span></span> <span data-ttu-id="4e737-198">대상이 너무 작으면 조작 경험을 피곤하고 당황스러운 것으로 설명합니다.</span><span class="sxs-lookup"><span data-stu-id="4e737-198">If targets become too small, users describe the experience as fatiguing and frustrating.</span></span>
